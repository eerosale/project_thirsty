The Problem

-Going to a bar during peak times =! fast service

-Hosting a party =! fun times

-Private bar tenders are expensive to hire.

-Drinks are not always consistently made.

-I am too lazy to make myself a drink.

Our Solution

We made an automous bar tender with this criteria:

  • Speed

    Four diaphragm pumps that can make a drink in < 30 seconds

  • Portability

    It is fairly easy to transport

  • Variety of Drink Options

    We chose four bottles for the system

  • Cost

    Entire project could be made for < $100

    • The Design

    Android App

    • Allows the user to connect to the proper Bluetooth module, HC-05.
    • Displays on and off buttons to manually control selected group.
    • Has List of available drinks to make.
    • Selected drink has description.

    Microcontroller

    • Receives Bluetooth messages
    • Decrypts messages and processes to respective functions

    System

    • Appropriate pumps and flowmeters dispense liquid
    • Hall-effect flowmeters send increment ticks back to microprocessor
    • Processor turns off pumps and flowmeters once appropriate liquid volume is reached

    Results

    Project Thirsty Achieved the Stated Design Goals:

    • Provides speed in dispensing the liquid
    • Materials used are cost effective
    • Has wireless user interface
    • The provided android application is easy to use
    • Choice of 4 bottles allows multiple drinks
    • Project is in a table top enclosure for transportation

    Future Work

    • Replacement of diaphragm pumps with rotary pumps
    • Smaller enclosure
    • Self enclosed battery system (one power source)
    • Have more than 4 bottles to make beverages
    • Make it wifi enabled and have a Web App
